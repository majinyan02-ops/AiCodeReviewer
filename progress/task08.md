  JavaParserService 能力

  ┌─────────────────────────────┬─────────────────────────────────┐
  │            方法             │              说明               │
  ├─────────────────────────────┼─────────────────────────────────┤
  │ scan(projectId, sourcePath) │ 扫描整个目录，返回 ScanContext  │
  ├─────────────────────────────┼─────────────────────────────────┤
  │ scanFile(filePath)          │ 扫描单个文件，返回 ScannedClass │
  └─────────────────────────────┴─────────────────────────────────┘

  ScanContext 扫描结果结构

  ScanContext
  ├── projectId          # 项目ID
  ├── rootPath           # 扫描根目录
  ├── controllers[]      # Controller 列表
  ├── services[]         # Service 列表
  ├── mappers[]          # Mapper 列表
  ├── entities[]         # Entity 列表
  ├── others[]           # 其他类
  ├── totalFiles         # 扫描文件数
  └── elapsedMs          # 耗时(ms)

  类类型识别策略（3 层匹配）

  1. 路径匹配：/controller/ → Controller, /service/ → Service
  2. 注解匹配：@RestController → Controller, @Service → Service
  3. 命名匹配：XxxController → Controller, XxxServiceImpl → Service

  扫描 API

  GET /api/scan/{projectId}    → 扫描 git-repos/{projectId}/ 下所有 Java 文件
